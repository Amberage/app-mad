Encapsulamiento

Retroalimentación: Actividad 1

OBJETIVOS

  • 1. Comprender el concepto de encapsulamiento en la programación orientada a objetos.
  • 2. Identificar situaciones del mundo real en las que el encapsulamiento es esencial.
  • 3. Evaluar si un escenario dado requiere encapsulamiento o no.

RETROALIMENTACIÓN

Escenario 1: Aplicación de Gestión de Correos Electrónicos

Problema: En una aplicación de gestión de correos electrónicos, es necesario rastrear la dirección de correo electrónico de un usuario.

De acuerdo con el problema planteado, ¿es necesario aplicar encapsulamiento a "correoElectronico"?

En este escenario SI es necesario el encapsulamiento, pues la dirección de correo electrónico es información privada y sensible. El encapsulamiento es necesario para proteger la privacidad del usuario y garantizar que solo se acceda y modifique de manera autorizada.

private String correoElectronico;

Escenario 2: Sistema de Registro de Eventos Públicos

Problema: En un sistema que registra eventos públicos, como conferencias o conciertos, es necesario rastrear la fecha de cada evento.

¿En relación con el problema planteado, se debe encapsular la variable "fechaEvento"?

En este escenario NO es necesario el encapsulamiento, pues la fecha y hora del evento suelen ser información que se divulga ampliamente y no necesita protección especial. En este caso, el acceso público a esta información es apropiado y no hay necesidad de ocultarla.

public Date fechaEvento;

Escenario 3: Sistema de Control de Inventario en una Tienda de Ropa

Problema: En un sistema de control de inventario en una tienda de ropa, es necesario llevar un registro de la cantidad de existencias de cada artículo de ropa.

¿Según el problema planteado, es necesario encapsular la variable "cantidadExistencias"?

En este escenario SI es necesario el encapsulamiento, pues la cantidad de existencias afecta la disponibilidad de productos para los clientes y la gestión de pedidos de reposición. El encapsulamiento es necesario para evitar que las existencias se actualicen de manera incorrecta y para garantizar un control adecuado del inventario.

private int cantidadExistencias;

Escenario 4: Sistema de Control de Temperatura en una Industria de Alimentos

Problema: En un sistema de control de temperatura en una fábrica de alimentos, es fundamental rastrear la temperatura de almacenamiento de un producto.

¿De acuerdo con el problema planteado, es necesario encapsular la variable "temperaturaAlmacenamiento"?

En este escenario SI es necesario el encapsulamiento, pues la temperatura de almacenamiento es crítica para garantizar la seguridad y la calidad de los alimentos. El encapsulamiento es necesario para controlar la temperatura de manera precisa y evitar cambios que puedan comprometer la calidad del producto.

private double temperaturaAlmacenamiento;

Escenario 5. Clase de Constantes Matemáticas

Problema: Suponga que tiene una clase donde las variables son constantes de matemáticas, por ejemplo, PI.

De acuerdo con el problema planteado, analiza si las constantes de matemáticas necesitan encapsulamiento o no

En este escenario NO es necesario el encapsulamiento, pues las constantes matemáticas son valores conocidos y públicos que no deben cambiar. No hay necesidad de encapsular estas constantes, ya que se espera que sean accesibles públicamente.

public static final double PI = 3.14159265359;

Escenario 6. Sistema de Registro de Calificaciones en una Escuela

Problema: En un sistema de registro de calificaciones escolares, se necesita rastrear las calificaciones de un estudiante en una asignatura.

De acuerdo con el problema planteado, analiza si una calificación necesita de encapsulamiento o no.

En este escenario SI es necesario el encapsulamiento, pues las calificaciones son datos sensibles y deben estar encapsuladas para garantizar la precisión y evitar la manipulación no autorizada.

private double calificacion;

Escenario 7. Sistema de Configuración de Software

Problema: En un sistema de configuración de software, se almacenan opciones de configuración, como el idioma, que los usuarios pueden modificar según sus preferencias.

De acuerdo con el problema planteado, ¿es necesario encapsular la variable "configuracionIdioma"?

En este escenario NO es necesario el encapsulamiento, pues las opciones de configuración son datos que los usuarios esperan modificar fácilmente. No es necesario encapsular esta información, ya que se supone que debe ser accesible y modificable públicamente.

public String configuracionIdioma;

Escenario 8. Sistema de Registro de Eventos de Uso de Aplicaciones

Problema: En un sistema de registro de eventos de uso de aplicaciones como los registros de eventos y las acciones realizadas por los usuarios es indispensable tener acceso para el análisis.

De acuerdo con el problema planteado, ¿es necesario encapsular el registro de eventos?

En este escenario NO es necesario el encapsulamiento, pues los registros de eventos son esenciales para comprender el comportamiento de las aplicaciones y solucionar problemas. No es necesario ocultarlos.

public List<Evento> registroEventos;
